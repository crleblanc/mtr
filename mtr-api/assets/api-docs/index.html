MTR API

Welcome to the MTR API.

The GeoNet project makes all its data and images freely available. Please ensure you have read and understood our Data Policy and Disclaimer before using any of these services.

Endpoints

The following endpoints are available:

  • App - Find applications.
  • App Metric - application metrics.
  • Application Counter - application counters.
  • Application Metric - application metrics.
  • Application Timer - application timers.
  • Data Completeness - completeness for data.
  • Data Completeness Summary - summary of data completeness.
  • Data Completeness Tag - tag data completeness metrics.
  • Data Completeness Type - types for data completeness.
  • Data Latency - latency for data.
  • Data Latency Summary - summary for data latency.
  • Data Latency Tag - tag data latency metrics.
  • Data Latency Threshold - set thresholds on data latency.
  • Data Site - sites for data.
  • Data Type - types for data.
  • Field Device - field devices.
  • Field Metric - field metrics.
  • Field Metric Summary - Field metric summaries.
  • Field Metric Tag - tags for field metrics.
  • Field Metric Threshold - thresholds for field metrics.
  • Field Model - models for field devices.
  • Field State - state for field devices.
  • Field State Tag - tags can be added to field state.
  • Field Type - field metric types.
  • Tag - find tags.
  • Tag - Tags can be added to metrics.

All requests should be made over HTTPS.

Versioning

API queries may be versioned via the Accept header. Please specify the Accept header for your request exactly as specified for the endpoint query you are using.

If you don't specify an Accept header with a version then your request will be routed to the current highest API version of the query or the default route.

Compression

The response for a query can be compressed. If your client can handle a compressed response then the reduced download size is a great benifit. Gzip compression is supported. You can request a compressed response by including gzip in your Accept-Encoding header.

Bugs

The code that provide these services is available at https://github.com/GeoNet/mtr If you believe you have found a bug please raise an issue or pull request there. Alternatively contact us detailing the issue.
